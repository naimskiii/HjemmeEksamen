Commit History and Documentation

Commit history is one of the most crucial aspects of collaboration in GitHub. By recording each small change in a commit, developers create a clear trail of modifications. These frequent commits serve as checkpoints, allowing the team to trace back changes if an issue arises and understand the rationale behind each change. Small, incremental commits are particularly helpful as they make it easier to pinpoint when and where an issue occurred, rather than having to navigate large, infrequent commits. As seen in the examples provided, frequent and clear commits with descriptive messages contribute to smoother project management and effective documentation.

What was the challanges with using github

Throughout the process, there were a couple challenges. We encountered a few errors, especially with pushing and pulling changes to and from the repository. The conflicts arose mostly when multiple people made changes to the same part of the project, but these were resolved through clear communication and team effort. Another thing was that sometimes we deleted each other's part by accident and we managed to add and fix this with good communication and collaboration throughout the process. GitHub's version control system allowed us to track the changes, which made it easier to identify and fix any issues.
